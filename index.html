1. Topic

This project focuses on the Mausoleum of Khoja Ahmed Yasawi, a UNESCO World Heritage Site located in Turkistan, Kazakhstan. The goal is to explore how its Wikidata representation can be enriched with religious, cultural, and pilgrimage-related knowledge.

Wikidata Entry Wikipedia Article UNESCO Page
Research Question: How can Wikidata for the Mausoleum of Khoja Ahmed Yasawi be enriched through RDF triples and knowledge graph modeling?

2. SPARQL Exploration

We used Wikidata Query Service to explore the current RDF description of the Mausoleum and identify missing semantic relations.

Query 1 — Existing Wikidata Properties

SELECT DISTINCT ?propertyLabel ?valueLabel
WHERE {
  wd:Q46069 ?prop ?value .
  ?property wikibase:directClaim ?prop .

  SERVICE wikibase:label {
    bd:serviceParam wikibase:language "en" .
  }
}
ORDER BY ?propertyLabel
LIMIT 200

Query 2 — Checking Yasawiyya Connection

SELECT ?propertyLabel ?valueLabel
WHERE {
  wd:Q46069 ?prop ?value .
  ?property wikibase:directClaim ?prop .

  FILTER(CONTAINS(LCASE(STR(?valueLabel)), "yas"))

  SERVICE wikibase:label {
    bd:serviceParam wikibase:language "en" .
  }
}

Query 3 — Checking Pilgrimage Function

SELECT ?propertyLabel ?valueLabel
WHERE {
  wd:Q46069 ?prop ?value .
  ?property wikibase:directClaim ?prop .

  FILTER(CONTAINS(LCASE(STR(?valueLabel)), "pilgrim"))

  SERVICE wikibase:label {
    bd:serviceParam wikibase:language "en" .
  }
}

Query 4 — UNION Query for Related Names

SELECT DISTINCT ?item ?itemLabel
WHERE {
  {
    ?item rdfs:label ?itemLabel .
    FILTER(REGEX(?itemLabel, "Yasawi", "i"))
  }
  UNION
  {
    ?item rdfs:label ?itemLabel .
    FILTER(REGEX(?itemLabel, "Arystan|Arslan Bab", "i"))
  }

  SERVICE wikibase:label {
    bd:serviceParam wikibase:language "en" .
  }
}
LIMIT 50

Query 5 — OPTIONAL Query

SELECT DISTINCT ?item ?itemLabel ?locationLabel
WHERE {
  ?item wdt:P31 wd:Q41176 .

  OPTIONAL {
    ?item wdt:P276 ?location .
  }

  SERVICE wikibase:label {
    bd:serviceParam wikibase:language "en" .
  }
}
LIMIT 50

3. Knowledge Gaps

Gap 1

Missing connection with the Yasawiyya Sufi Order.

The Wikidata entry does not explicitly connect the mausoleum to the Yasawiyya tradition.

Gap 2

Missing representation as a pilgrimage site.

The spiritual function of the mausoleum as a pilgrimage destination is underrepresented.

Gap 3

Missing spiritual and religious significance.

The entry contains basic historical data, but lacks deeper religious context.

Gap 4

Missing links with related religious heritage sites.

The mausoleum is not clearly connected to sites such as Arystan Bab Mausoleum.

4. LLM Evaluation

We compared ChatGPT and Gemini using three prompting strategies: Zero-shot, Few-shot, and Chain-of-Thought.

Prompt Type ChatGPT Gemini Human Assessment
Zero-shot Generated clear triples about Yasawiyya, pilgrimage, Sufism, and heritage sites. Generated similar triples with more detailed explanations. Both models were relevant. ChatGPT was clearer; Gemini was richer.
Few-shot Followed the example structure closely and stayed focused. Adapted strongly to the example and created more complex semantic relations. Few-shot improved both models. Gemini benefited more from the example.
Chain-of-Thought Identified the same four gaps found through SPARQL exploration. Expanded the analysis with additional ideas such as Taykazan and architectural influence. ChatGPT stayed aligned with the project; Gemini was more exploratory.

5. RDF Enrichment

Based on SPARQL exploration and LLM outputs, we propose the following RDF triples: 

:Mausoleum_of_Khoja_Ahmed_Yasawi :associatedWith :Yasawiyya_Sufi_Order .

:Mausoleum_of_Khoja_Ahmed_Yasawi :hasFunction :Pilgrimage_Site .

:Mausoleum_of_Khoja_Ahmed_Yasawi :associatedWith :Sufism .

:Mausoleum_of_Khoja_Ahmed_Yasawi :relatedHeritageSite :Arystan_Bab_Mausoleum .

These triples enrich the knowledge graph by representing religious traditions, pilgrimage practices, spiritual significance, and connections with other heritage sites.

6. Challenges

SPARQL Complexity

We had to learn how to use FILTER, REGEX, OPTIONAL, UNION, LIMIT, and ORDER BY.

Identifying Real Gaps

Some information existed in Wikipedia but was not represented as structured data in Wikidata.

LLM Limitations

Gemini sometimes generated broader suggestions that moved beyond our research focus.

Human Evaluation

We manually compared outputs to select the most useful RDF triples.

7. Conclusion

This project demonstrated how Large Language Models can support knowledge graph enrichment for cultural heritage.

By analyzing the Wikidata entry of the Mausoleum of Khoja Ahmed Yasawi, we identified several gaps related to religious significance, pilgrimage practices, and connections with other heritage sites.

ChatGPT and Gemini were evaluated through Zero-shot, Few-shot, and Chain-of-Thought prompting. ChatGPT produced more focused outputs, while Gemini generated broader and more exploratory suggestions.

The proposed RDF triples improve the semantic richness of the knowledge graph by adding cultural, religious, and pilgrimage-related information.
